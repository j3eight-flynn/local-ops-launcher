# Asset Forge Style Bible

## Project Name

Unconfigured project. Run `asset-forge setup` inside the target project.

## Project Purpose

Create project-specific visual assets that can be reviewed, approved, and used by the local application.

## Audience

Defined during setup.

## Emotional Target

Defined during setup.

## Genre

Defined during setup.

## Visual Style

Distinctive, consistent, and project-owned. Use broad art-direction language instead of copying a living artist, franchise, protected character, commercial logo, or existing game art.

## Realism Level

Defined during setup.

## Color Palette

Defined during setup.

## Texture Rules

Defined during setup.

## Lighting Rules

Use intentional lighting that supports the asset purpose and remains consistent across an asset pack.

## Composition Rules

Prioritize clear silhouettes, readable subjects, and safe space when the asset will sit under UI overlays.

## Character Design Rules

Keep anatomy, costume language, era, and materials consistent with the project setting.

## Environment Design Rules

Environments should communicate place, scale, story function, and atmosphere without adding random lore.

## UI Asset Rules

UI assets must preserve legibility and avoid baked-in text unless explicitly requested.

## Icon Rules

Icons should be recognizable at small sizes and use transparent backgrounds when configured.

## Card Art Rules

Card art should leave framing and rules text to the app unless the request explicitly asks for printed text.

## Background Art Rules

Backgrounds should preserve safe areas for characters, dialogue, overlays, or cards when required.

## Sprite Rules

Sprites should favor clear silhouettes, consistent scale, and transparent backgrounds.

## What To Avoid

Avoid generic AI art, fan art, protected IP, living artist imitation, watermarks, random logos, text artifacts, inconsistent style, and unrequested modern objects.

## Approved Reference Assets

Place approved references in `/public/assets/generated/references`.

## Examples of Strong Asset Prompts

Create a character portrait for the project using the configured genre, palette, lighting, composition, intended UI use, dimensions, and negative instructions.

## Examples of Bad Asset Prompts

Make cool art. Make it like a famous living artist. Make a character from an existing franchise.
